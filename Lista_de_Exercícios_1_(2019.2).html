Unidade 1.1: processos e thrads

  1. O que são processos?

    2. São instâncias de programas em execução.

  2. Que benefícios são proporcionados por processos?

    3. Um EEV (Espaço de Endereçamento Virtual) reservado para aquela instância do programa, dados sobre a utilização e referências a arquivos e dispositivos em uso. Processos permitem que o programador escreva programas se apoiando na ilusão de que ao ser executado disporá da memória, da CPU e do computador de forma exclusiva.

  3. Por que costumamos dizer que um processo é uma instância de um programa?

    4. Porque um mesmo programa pode ser utilizado em vários processos (várias instâncias). Cada aba de um browser é um processo separado, em vários browsers modernos.

  4. O que é o espaço de endereçamento virtual?

    5. A memória virtual utiliza abstração semelhante a um vetor em linguagens de alto nível, só que em relação aos endereços dos programas e dados. Um programa no ambiente de memória virtual não faz referência a endereços físicos de memória (endereços reais), mas apenas a endereços virtuais. No momento da execução de uma instrução, o endereço virtual referenciado é traduzido para um endereço físico, pois o processador manipula apenas posições da memória principal. O mecanismo de tradução do endereço virtual para endereço físico é denominado mapeamento.

  5. O que são threads?

    6. É como se um único processo se comportasse como vários processos paralelos que compartilham um mesmo espaço de endereçamento virtual. Trata-se de um mecanismo poderoso em diversas aplicações em que é necessário que várias coisas aconteçam ao mesmo tempo.

  6. Que comando unix permite consultar/listar os processos em execução?

    7. O comando top e o comando ps.

  7. Classifique as afirmações em Verdadeiras e Falsas. Justifique sua resposta.

    • a. Os processos em execução em uma máquina compartilham o Espaço de Endereçamento Virtual proporcionado pelo sistema operacional.

      • Se for cada processo com seu próprio EEV, verdadeiro.

    • b. O Espaço de Endereçamento Virtual é uma abstração provida pelo sistema operacional que permite a um processo atuar como se tivesse uma memória de uso exclusivo.

      • Verdadeiro.

    • c. É o uso de múltiplas CPUs (computadores multi-core) que permite o uso de threads e, portanto, de sistemas multi-thread.

      • Falso. Threads são executadas (geralmente) em um único processador e funcionam normalmente como um 'falso paralelismo' entre atividades de um processo.

    • d. É o uso de múltiplas CPUs (computadores multi-core) que permite que o computador execute vários processos simultaneamente.

      • Falso. Processos não executam simultaneamente, mas sim com um tempo de respostas tão rápido provido pelo hardware moderno que o usuário tem a impressão de paralelismo nas execuções. Além disso, não são necessárias várias CPUs para executar vários processos.

    • e. É graças ao conceito de threads que um sistema operacional pode executar o mesmo programa múltiplas vezes e simultaneamente.

      • Falso. Um programa pode ser executado múltiplas vezes sem a utilização de threads. Threads possibilitam uma certa funcionalidade ser executada 'ao mesmo tempo' que outra em um mesmo processo e utilizando o mesmo EEV.

    • f. Threads são usadas nos servidores para permitir que um mesmo processo atenda a múltiplas requisições simultaneamente.

      • Verdadeiro.

Unidade 1.2: streams e pipes

  1. O que são streams?

    2. Fluxos de dados sequencial.

  2. Quais os streams default de um processo?

    3. STDIN, STDOUT e STDERR. Todo processo Unix é, ao mesmo tempo, produtor e consumidor de streams.

  3. Para que devem ser usados stdin, stdout e stdev?

    4. Entrada padrão de dados, saída padrão de dados e saída padrão de erros.

  4. O que é redirecionamento?

    5. O mecanismo que SOs usam para conectar um processo a um arquivo.

  5. O que é um pipeline?

    6. Uma sequencia de conexões entre streams de processos via pipes pelo SO.

  6. Qual a diferença entre o shell e um terminal?

    7. Shell é o interpretador dos comandos que são escritos pelo usuário através de um terminal. Terminal é uma interface de utilização textual que possibilita a comunicação com o Shell.

  7. Que comando pode ser usado para listar os processos em execução de forma que a lista resultante seja armazenada no arquivo arq.txt?

    8. top > processos.txt

  8. Um processo pode estar conectado a múltiplos streams simultaneamente?

    9. Sim.

  9. Como o sistema operacional executa um pipeline unix? Cada programa é executado na ordem indicada pelo pipeline, coletando a saída de cada um deles e armazenando-a para que seja usada como entrada do próximo programa? Explique.

Unidade 1.3: Sockets

  1. O que são sockets?

    2. São como pipelines porém para conexão entre máquinas.

  2. O que são conexões TCP e quais suas características relevantes?

    3. Protocolo de transmissão de dados. A versatilidade e robustez do TCP tornou-o adequado a redes globais, já que este verifica se os dados são enviados de forma correta, na sequência apropriada e sem erros, pela rede. Ponto a ponto - uma conexão TCP é estabelecida entre dois pontos. Entrega ordenada - A aplicação faz a entrega ao TCP de blocos de dados com um tamanho arbitrário num fluxo (ou stream) de dados. Controle de fluxo. Controle de congestionamento.

  3. O que se quer dizer com “conexões TCP são full-duplex”?

    4. É possível a transferência simultânea em ambas direções (cliente-servidor) durante toda a sessão. Apesar disso, em alguns momentos, o protocolo necessita que algum pacotes de dados cheguem para que se dê o envio de outros, o que limita as transmissões.

  4. A comunicação por TCP é baseada em requisições e respostas? Explique.

    5. Acho que sim.

  5. O que é um endereço TCP? Como é formado?

    6. Endereço IP é um endereço virtual que representa a numeração do host.

  6. O que se quer dizer com “_conexões TCP transmitem bytes, não caracteres”?

    7. Porque TCP transmite os dados de forma binária.

  7. Usando o comando nc, indique como criar um servidor que “ouça” na porta 2626 da máquina local.

    8. O comando nc -l -p 2626

  8. Usando o comando nc, indique como criar um cliente para se conectar em um servidor na porta 2626 da máquina local.

    9. nc 2626 2626

  9. Escreva um programa em python que a partir de um endereço dado pelo usuário, estabeleça uma conexão TCP e que envie uma sequência de 100 números aleatórios entre 0 e 200.

    10. Perai parcero.

  10. Reescreva o programa da atividade anterior, mudando a codificação utilizada. Se você os números foram enviados como strings, desta vez represente-os como bytes (observe que isto é possível, porque os valores estão entre 0 e 200). Se na atividade anterior você já havia utilizado bytes, desta vez, represente os números em formato textual. Observe que você precisará de um separador entre as representações neste caso. Sugestão: use \n.
